**Description of Each Item**

**SKILL.md (master instructions)** The entry point Claude reads first. Defines what this skill does, when to invoke it, the overall workflow (intake → audit → strategy → execution → measurement), and how to navigate to the right reference file based on what the user needs. Includes the "if user says X, read file Y" routing logic.

**references/philosophy.md** The master frame for how to think about SEO. Core principles like "rank for intent, not just keywords," "Google rewards the best answer to a query," "SEO is a compounding asset, not a campaign," and "humans first, crawlers second." Sets the worldview before tactics.

**references/technical-foundation.md** The pre-launch checklist. HTTPS, mobile-friendliness, page speed, XML sitemaps, robots.txt, URL structure, canonical tags, hosting choices, Core Web Vitals. Everything that has to be right before any content strategy matters.

**references/keyword-research.md** Full process for finding the right keywords. Seed keyword brainstorming, search intent classification (informational/navigational/commercial/transactional), competition assessment, long-tail strategy, free vs. paid tools, building topic clusters from research.

**references/on-page-seo.md** How to optimize a single page. Title tag formulas, meta description writing, H1/H2/H3 hierarchy, keyword placement, image alt text, internal link placement within content, URL slug optimization, content length guidance.

**references/content-strategy.md** The architectural plan for what to publish. Pillar page + cluster model, content types (how-to, listicle, comparison, ultimate guide, case study), publishing cadence, content calendars, evergreen vs. timely content, identifying content gaps.

**references/content-writing.md** How to actually write the content. Hooks, scannability, formatting for both readers and featured snippets, answering search intent fully, satisfying People Also Ask boxes, writing meta-friendly intros, balancing depth with readability.

**references/internal-linking.md** Site architecture and how link equity flows. Hub-and-spoke structures, anchor text strategy, contextual vs. navigational links, orphan page prevention, the "three clicks from homepage" rule, breadcrumbs, related content sections.

**references/schema-markup.md** Structured data for rich results. Article, Book, Product, Organization, FAQ, HowTo, Review, BreadcrumbList schemas. JSON-LD examples ready to paste. When each type triggers rich results in search.

**references/link-building.md** Earning external backlinks. Guest posting, digital PR, broken link building, resource page outreach, HARO/Qwoted responses, podcast appearances, partnership links. What to avoid (PBNs, link schemes, paid links).

**references/local-seo.md** For businesses with physical locations or service areas. Google Business Profile optimization, NAP consistency, local citations, review generation strategy, local keyword targeting, geo-targeted landing pages.

**references/technical-audits.md** Diagnosing problems on existing sites. Crawl errors, index bloat, duplicate content, thin content, broken links, redirect chains, mobile usability issues. Tools to use, what to fix first, severity prioritization.

**references/analytics-tracking.md** Setting up measurement. GA4 configuration, Search Console setup, conversion tracking, key metrics (impressions, CTR, average position, organic sessions, conversions), how to read reports, what's signal vs. noise.

**references/ai-content-guidelines.md** Using AI to scale without triggering Google's spam policies. The Helpful Content Update implications, the "human-in-the-loop" requirement, when AI is fine vs. when it kills rankings, editing AI output to add genuine value.

**references/e-e-a-t.md** Experience, Expertise, Authoritativeness, Trustworthiness - Google's quality framework. Author bios, credentials display, citations, original research, first-hand experience signals, building topical authority systematically.

**references/algorithm-updates.md** Staying resilient. How core updates work, what types of sites get hit, recovery patterns, what to do when traffic drops, building an SEO strategy that doesn't depend on a single tactic.

**templates/intake-questionnaire.md** Questions Claude asks the user before building any SEO plan. Business model, target audience, current traffic, competitors, conversion goals, available resources, technical constraints, timeline expectations.

**templates/keyword-research-template.md** The structure for organizing a keyword research output. Columns for keyword, search volume, difficulty, intent type, target page, priority. Ready format for the user to fill or for Claude to populate.

**templates/content-brief-template.md** The blueprint Claude uses when writing or planning any single piece of content. Target keyword, search intent, word count target, required sections, internal links to include, schema to apply, CTA placement.

**templates/seo-audit-template.md** The full diagnostic checklist for assessing any existing site. Technical issues, on-page issues, content gaps, backlink profile, competitive position. Produces a prioritized fix list.

**templates/90-day-roadmap.md** Week-by-week action plan from launch to first results. Days 1-30 (foundation), 31-60 (content velocity), 61-90 (link building + optimization). Specific tasks, not generic advice.

**templates/monthly-report-template.md** The reporting structure for tracking SEO progress over time. Traffic trends, ranking changes, top-performing content, new keywords gained, conversions from organic, recommendations for next month.

**examples/christian-library-example.md** Your TeamSuccessNetwork case fully worked out as a reference. 200+ books, 30 categories, free HTML content, reader-retention focus. Shows the full skill applied to a real situation, including specific keyword sets and the reader-pathway content strategy we discussed.

**examples/ecommerce-store-example.md** A different application - product-focused SEO with category pages, product pages, and review-driven content. Shows how the same principles flex for transactional intent.

**examples/local-service-example.md** A third application - service business with geographic targeting. Demonstrates local SEO, Google Business Profile work, and city-specific landing pages. Rounds out the range of use cases.
